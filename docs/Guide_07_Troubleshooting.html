🚫 Problem 1: CORS Errors
Access to XMLHttpRequest at 'https://api.snapalign.in' from origin 'https://snapalign.in' has been blocked by CORS policy: No 'Access-Control-Allow-Origin' header is present.
🔴 What is CORS?

CORS = Cross-Origin Resource Sharing. It's a security feature that stops websites from talking to servers they shouldn't.

Think of it like: Your frontend (Vercel) trying to call your backend (Render), but the backend says "I don't know you!" 🚫

✅ Solutions:

1
Check Backend CORS Settings

Open /app/backend/server.py and find:

app.add_middleware( CORSMiddleware, allow_origins=["*"], # Should be "*" or your domain allow_credentials=True, allow_methods=["*"], allow_headers=["*"], )
2
Check Render Environment Variable

Go to Render Dashboard → Environment → Check CORS_ORIGINS

CORS_ORIGINS=*

Or for specific domains:

CORS_ORIGINS=https://snapalign.in,https://www.snapalign.in
3
Restart Backend

After changing environment variables, click "Manual Deploy" in Render to restart.

✅ Verified Working: If CORS_ORIGINS=* is set, any frontend can connect.
🔒 Problem 2: MongoDB Connection Failed
pymongo.errors.ServerSelectionTimeoutError: connection closed, Timeout: 30s, Topology Description: ...
🔴 What's Happening?

MongoDB Atlas blocks connections from unknown IP addresses. Render's IP changes, so MongoDB doesn't recognize it.

Think of it like: A bouncer at a club who doesn't have your name on the list! 🚪

✅ Solution: Whitelist All IPs

1
Go to MongoDB Atlas

Login at cloud.mongodb.com

2
Navigate to Network Access

Left sidebar → Security → Network Access

3
Add IP Address

Click "Add IP Address" → "Allow Access from Anywhere"

IP: 0.0.0.0/0 Comment: Allow all (for Render)
4
Wait 1-2 Minutes

Changes take a moment to propagate.

⚠️ Security Note: Allowing all IPs is less secure but necessary for services like Render with dynamic IPs. Your database is still protected by username/password!
🔗 Problem 3: Invalid Connection String
pymongo.errors.ConfigurationError: Invalid URI host: password contains unescaped characters
🔴 What's Happening?

Your MongoDB password has special characters (like @, #, %) that need to be "escaped" (converted to safe codes).

✅ Solution: URL-Encode Special Characters

Character Replace With
@%40
#%23
$%24
%%25
&%26
+%2B
/%2F

Example:

# Password: My@Pass#123 # Encoded: My%40Pass%23123 # Full connection string: mongodb+srv://user:My%40Pass%23123@cluster.mongodb.net/snapalign
💡 Pro Tip: Use a simple password without special characters to avoid this issue!
😴 Problem 4: Backend is Sleeping (Slow First Request)
🔴 Symptoms
  • First page load takes 30+ seconds
  • Subsequent requests are fast
  • Happens after ~15 minutes of no activity

✅ Solutions:

1
Option A: Upgrade to Paid Plan ($7/month)

Render Starter plan keeps your service always running.

2
Option B: Use UptimeRobot (Free)

Set up a free ping service to keep your backend awake:

  1. Go to uptimerobot.com
  2. Create free account
  3. Add new monitor: HTTP(s)
  4. URL: https://snapalign-backend.onrender.com/api/health
  5. Interval: 5 minutes
✅ UptimeRobot will ping your backend every 5 minutes, keeping it awake!
